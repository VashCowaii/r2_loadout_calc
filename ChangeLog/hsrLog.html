Nov 21th, 2025

  • Added the ability to allow 2pc set mixing on the body relics, for each character individually. This also moves the settings toggle into Occlusion filters for each respective character on the team.
    • This does mean that the settings entry in search settings has been removed, no need for a team-based toggle on that setting when it exists for each character.
  • Added a check that will kill any query before it can launch, if the user has defined minimum substat roll values on character locks, that together exceed the total added rolls possible per character.
    • When this check triggers, you'll be met with a pop-up that will notify you of which character or characters are the ones that have too much on their minimum rolls.
  • Added the ability to view Action Counts for each character in Analytics -> Summary, next to the various assorted totals for other things.
  • Adjusted the page-load character config to no longer populate Saber at e6, or Sunday at e1. She was only there for testing purposes, obv you should be defaulting to e0 on a 5star character on page load.
    • I will clarify that whenever you change a character slot to a new 5star character, it will always default to e0, the only reason this is different is because it was from the page-load character config.
  • Added a healing icon to any healing actions that take place in an expanded action view. Similar to buffs, but instead of an up arrow or down arrow when a buff is removed, it will instead show the healing + sign between the healing source and the healing target.
  • Fixed an issue where duplicate substats on a relic didn't always show an error when they should have, telling the user that duplicate substats are obv not good.
  • Fixed an issue where when changing the mainstat on a relic it would subvert some error message displays that should have displayed.
  • Fixed an issue where if a substat had a conflict with a relic's mainstat, the error wouldn't display that would tell people that shouldn't be a thing.
  • Fixed an issue where if a Harmony character used "But the Battle Isn't Over"(Bronya sig), it would try to apply its post-skill buff to non-buffable entities(summons, that aren't memosprites)
  • Fixed an issue where Bronya's skill advance could improperly cache who character 1 was in the team for the sake of targeting and actually applying the advance. The result was that sometimes the advance wouldn't apply.
  • Fixed an issue where this changelog was showing TFD as the header value on a pageload instead of HSR.
